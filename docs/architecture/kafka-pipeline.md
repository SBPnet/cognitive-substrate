# Kafka Pipeline Architecture

## Implementation Status

The source-of-truth runtime topic registry is `packages/kafka-bus/src/topics.ts`. Any topic name that appears only in older design documents is a proposal until it is added there and wired through a producer or consumer.

## Purpose

Kafka is the cognitive signal bus. It separates fast user-facing interaction, memory lifecycle events, agent reasoning, policy updates, consolidation, telemetry ingestion, operational primitive detection, and reinforcement feedback.

## Runtime Topics

The implemented registry groups topics by namespace:

- `experience.*` for raw and enriched experience events.
- `memory.*` for indexed and semantic memory lifecycle events.
- `agent.*` for multi-agent reasoning request and response flow.
- `worldmodel.*` for prediction events.
- `policy.*` for evaluation and update events.
- `goal.*` for long-horizon goal progress.
- `consolidation.*` for offline replay requests.
- `identity.*` and `selfmod.*` for identity and self-modification events.
- `audit.*` for immutable audit mirroring.
- `interaction.*` for BFF response routing.
- `telemetry.*` and `cognition.*` for operational intelligence streams.

## Flow

Experience ingestion begins on `experience.raw`, enriches events, and confirms persistence on `memory.indexed`. Agent processing uses `agent.reasoning.request`, `agent.reasoning.response`, and `interaction.response`. Consolidation consumes `consolidation.request` and emits `memory.semantic.updated`. Policy and reinforcement flows use `policy.evaluation`, `policy.updated`, and operational cognition topics where worker wiring exists.

## Operational Topics

Telemetry topics are declared as diskless-capable high-volume streams in `packages/kafka-bus/src/topics.ts`. They are appropriate for raw metrics, logs, metadata, traces, normalized telemetry events, operational primitives, patterns, anomalies, and recommendations. Worker support is incremental and should be verified against `apps/workers/` before making runtime claims.

## Evidence

Topic names, partition counts, retention, replication factor, and diskless flags are declared in `packages/kafka-bus/src/topics.ts`. Behavioral evidence lives in worker tests, smoke scripts, and any report generated by `scripts/smoke/deep-smoke.ts`.
